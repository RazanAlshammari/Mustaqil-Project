───────────────  الملفات الأساسية (مطلوبة)  ───────────────

  app_v2.py                → التطبيق كامل (12 تبويباً + 4 نماذج)
  db.py                    → طبقة قاعدة البيانات (SQLite محلية)
  mustaqil_dataset_v2.xlsx → بيانات البذر الأولية (شيتان: شهري + مشاريع)
  fonts/                   → خط ثمانية (Thmanyah Sans) بصيغة woff2
  requirements.txt         → المكتبات المطلوبة

  هذه الملفات تكفي للتشغيل: app_v2.py + db.py + mustaqil_dataset_v2.xlsx + fonts/

  عند أول تشغيل يُنشئ db.py ملف mustaqil.db تلقائياً ويزرع فيه بيانات
  الإكسل. لا حاجة لأي إعداد يدوي لقاعدة البيانات.

───────────────  ملفات مساعدة (اختيارية)  ───────────────

  run_models_v2.py      → يشغّل النماذج الأربعة ويطبع نتائجها في الترمنال
  derive_weights_v2.py  → يثبت أن أوزان مؤشر الجدارة مشتقّة من البيانات
  generate_data_v2.py   → الكود الذي ولّد بيانات البذر (لو أردت تغييرها)
  monthly_data_v2.csv   → نسخة CSV من الشيت الشهري
  projects_data_v2.csv  → نسخة CSV من شيت المشاريع

───────────────  طريقة التشغيل محلياً  ───────────────

  1) ضع app_v2.py و db.py و mustaqil_dataset_v2.xlsx ومجلد fonts/ في مجلد واحد
  2) في الترمنال:   pip install -r requirements.txt
  3) شغّل التطبيق:  python -m streamlit run app_v2.py
  4) يفتح المتصفح تلقائياً على localhost:8501

  لرؤية نتائج النماذج في الترمنال:
       python run_models_v2.py

───────────────  النشر المجاني (Streamlit Community Cloud)  ───────────────

  1) أنشئ مستودع GitHub فارغاً وارفع محتويات هذا المجلد إليه (git init،
     git add، git commit، git remote add origin، git push)
  2) افتح share.streamlit.io وسجّل الدخول بحساب GitHub
  3) اضغط "New app"، اختر المستودع والفرع، وحدّد app_v2.py كملف التشغيل
  4) اضغط Deploy، يجهَز رابط عام خلال دقائق مثل: mustaqil.streamlit.app

  ملاحظة مهمة عن قاعدة البيانات على الاستضافة المجانية: قرص Streamlit
  Community Cloud مؤقّت. أي نشر جديد أو نوم التطبيق من عدم الاستخدام يعيد
  بناء الحاوية من مستودع GitHub، فتُفقد أي بيانات كُتبت أثناء التشغيل
  (أهداف جديدة، فواتير) ويُعاد زرع البيانات الأساسية فقط من ملف الإكسل.
  هذا مقبول تماماً لعرض حيّ في الهاكاثون. للاستخدام الإنتاجي الدائم
  الخطوة التالية هي الانتقال لقاعدة بيانات مُدارة مجانية مثل Supabase
  (Postgres)، ببساطة استبدال دوال db.py بنفس التوقيعات.

───────────────  الجديد في v2  ───────────────

  ✦ تبويب «مؤشر الجدارة»       — درجة ائتمانية من 100 + وثيقة دخل
  ✦ تبويب «كاشف العميل الخطر»  — يتنبأ بتأخّر السداد (نموذج رابع)
  ✦ تبويب «حاسبة الزكاة»       — نصاب 85 جم ذهب + 2.5%
  ✦ أوزان الجدارة مشتقّة من البيانات لا من الاجتهاد
  ✦ إصلاح تسريب بيانات في نموذج التسعير
  ✦ تحسين نموذج الدخل بميزات موسمية دورية
  ✦ تبويب «محاكي ماذا لو»       — أثر تغيير السعر/العملاء/الساعات
  ✦ تبويب «أشجار الأهداف»      — الادخار كقصّة بصرية تنمو
  ✦ معايرة البيانات: مجموع المشاريع = مجموع الدخل (كان متناقضاً)

───────────────  الجديد في v2.1  ───────────────

  ✦ قاعدة بيانات SQLite حقيقية (db.py) بدل القراءة المباشرة من الإكسل،
    وأهداف/فواتير المستخدم محفوظة فيها لا في ذاكرة الجلسة فقط
  ✦ تبويب «التمويل الإسلامي»    — يحوّل درجة الجدارة إلى منتج تمويلي
    فعلي (تورّق/إجارة/مرابحة/قرض حسن) بمبلغ وقسط وتصنيف ربح إرشادي
  ✦ تفسير كل توقّع فردي («أهم العوامل المؤثّرة») في تبويبات الجفاف
    والتسعير وكاشف العميل الخطر، لا مجرّد أوزان إجمالية
  ✦ مقارنة الأقران داخل اللوحة الرئيسية: ترتيبك المئوي بين فريلانسرز
    نفس تخصّصك في الدخل والاستقرار وربحية الساعة
  ✦ تبويب «الفاتورة الإلكترونية» — فاتورة ضريبية مبسّطة برمز QR بترميز
    TLV القياسي المعتمد في فوترة (ZATCA) المرحلة الأولى، مع سجلّ فواتير
  ✦ خط ثمانية (Thmanyah Sans) مضمَّن بالكامل بديلاً عن Tajawal، في
    الواجهة وكل مستندات PDF المُصدَّرة، بلا اعتماد على إنترنت

───────────────  نتائج النماذج الأربعة  ───────────────

  1) التنبؤ بشهر الجفاف     : دقة 89.7%
  2) التنبؤ بالدخل القادم   : R² = 0.132   (منخفض بطبيعته — الدخل عشوائي)
  3) اقتراح تسعير المشروع   : R² = 0.765
  4) كاشف العميل الخطر      : دقة 80.1% · AUC = 0.887

  ملاحظة: الأرقام قد تتغيّر ±1% حسب نسخة scikit-learn عندك.
  شغّل run_models_v2.py واعتمد ما يظهر لديك.

───────────────  التبويبات الاثنتا عشرة  ───────────────

   1.  اللوحة الرئيسية      — الراتب الاصطناعي + مؤشر الاستقرار + مقارنة الأقران
   2.  مؤشر الجدارة         — درجة ائتمانية + وثيقة دخل قابلة للتحميل
   3.  التمويل الإسلامي     — منتج التمويل المناسب لدرجتك (تورّق/إجارة/مرابحة)
   4.  تحذير الجفاف         — توقّع الشهر القادم + تفسير التوقّع
   5.  محفظة المشاريع       — ربحية كل مشروع وعميل
   6.  حاسبة التسعير        — السعر العادل لمشروع جديد + تفسير التوقّع
   7.  كاشف العميل الخطر    — احتمال تأخّر السداد + تفسير التوقّع
   8.  محاكي ماذا لو        — جرّب قراراتك قبل اتّخاذها
   9.  أشجار الأهداف        — الادخار كقصّة بصرية، محفوظة في قاعدة البيانات
  10.  حاسبة الزكاة         — زكاة المال وفق النصاب
  11.  الفاتورة الإلكترونية — فاتورة مبسّطة برمز QR متوافق مع ZATCA
  12.  أداء النماذج         — دقة النماذج الأربعة بشفافية
